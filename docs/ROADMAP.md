# FlowMind Roadmap

> 一句话 + 优先级。背景与设计原则见 [PRD.md](./PRD.md)。
>
> **优先级**: 🔴 P0（必做，下个版本）· 🟠 P1（应做）· 🟡 P2（想做）· ⚪ P3（也许做）

---

## v0.3 — 让画图更强大

主题：把"文档里嵌图"的差异化优势打透。

| 状态 | 优先级 | 项 | 描述 |
| --- | --- | --- | --- |
| 📋 | 🔴 P0 | Mermaid 块 | 文本驱动的流程图 / 时序图 / 类图，覆盖 80% 的"我就想快画一个图"场景 |
| 📋 | 🔴 P0 | Excalidraw 块 | 手绘风白板，适合架构示意、头脑风暴草图 |
| 📋 | 🟠 P1 | Kanban 视图块 | 嵌在文档里的轻量看板，列 / 卡片 / 拖拽，不做 status 自动化 |
| 📋 | 🟠 P1 | 图表全屏编辑 | 任意 diagram block 双击进入全屏画布，编辑完回到文档 |
| 📋 | 🟡 P2 | 模板库 | 新建文档时选模板（OKR、读书笔记、PRD、复盘、SWOT…） |
| 📋 | 🟡 P2 | 图表导出 PNG / SVG | 单独导出 diagram block，不必整篇 PDF |
| 📋 | ⚪ P3 | 节点链接到文档 | mind map 节点可跳转到另一篇文档（双向链接的图形入口） |

---

## v0.4 — 让写作更智能

主题：AI 与"图 + 文"定位深度结合，不做又一个侧边栏对话框。

| 状态 | 优先级 | 项 | 描述 |
| --- | --- | --- | --- |
| 📋 | 🔴 P0 | `/ai` slash 命令 | 在编辑器内直接用 AI 生成、续写、改写、翻译 |
| 📋 | 🔴 P0 | 选中文本 → 一键转 Mind Map | 把段落 outline 抽出来变成思维导图块（差异化杀器） |
| 📋 | 🔴 P0 | 多 provider 配置 | OpenAI、Anthropic、本地 Ollama；Settings 页配 API key |
| 📋 | 🟠 P1 | 文档总结 | 顶栏一键生成 TL;DR / 摘要插入到顶部 |
| 📋 | 🟠 P1 | AI 续写 | 段落末尾 Tab 触发续写建议（类 GitHub Copilot） |
| 📋 | 🟡 P2 | 选中段落 → 转 Flowchart | 用 LLM 抽流程，自动建节点 + 连线 |
| 📋 | 🟡 P2 | Prompt 模板 | 用户自存常用 prompt（"改成更正式"、"补充例子"…） |
| 📋 | ⚪ P3 | 本地知识库问答 | 跨文档 RAG，用 sqlite-vss 或 sqlite-vec 做向量检索 |

---

## v0.5 — 让文档可管理

主题：文档数量从 < 20 涨到几百时仍然可用。

| 状态 | 优先级 | 项 | 描述 |
| --- | --- | --- | --- |
| 📋 | 🔴 P0 | 文件夹 / 标签 | 至少二选一，建议先做标签（更灵活） |
| 📋 | 🔴 P0 | 全文搜索 | SQLite FTS5，标题 + 正文联合索引 |
| 📋 | 🟠 P1 | 版本历史 | 每次保存留 snapshot（最近 50 个），diff 视图 |
| 📋 | 🟠 P1 | 收藏 / Pin | 主页置顶常用文档 |
| 📋 | 🟡 P2 | 最近访问列表 | 快速回到上次编辑 |
| 📋 | 🟡 P2 | 文档关联 | `[[wiki link]]` 语法 + 反向链接面板 |

---

## v0.6 — 让数据可信赖

主题：附件、备份、迁出能力。

| 状态 | 优先级 | 项 | 描述 |
| --- | --- | --- | --- |
| 📋 | 🔴 P0 | 附件上传 | 图片拖拽 / 粘贴，存 `/data/uploads`，链接走相对路径 |
| 📋 | 🔴 P0 | 整库导出 | 一键下载所有文档（zip 含 .md + 附件） |
| 📋 | 🟠 P1 | DOCX 导出 | 单文档导出为 .docx |
| 📋 | 🟠 P1 | 定时备份 cron | 内置 cron 把 /data 打包到本地目录或 S3 |
| 📋 | 🟡 P2 | 备份恢复 | 上传备份 zip 一键覆盖恢复 |
| 📋 | 🟡 P2 | 图床外置 | 可选 S3 / Cloudflare R2 作为附件存储 |
| 📋 | ⚪ P3 | 导入 Markdown / Notion / Obsidian | 批量从其它工具迁入 |

## v0.7+ — 让协作更顺手（如果用户需要）

主题：取决于"小团队 Beth"persona 的反馈是否强烈。**默认不优先**。

| 状态 | 优先级 | 项 | 描述 |
| --- | --- | --- | --- |
| 📋 | 🟠 P1 | 协作者光标 + 头像 | y-awareness 已经支持，前端把它画出来 |
| 📋 | 🟠 P1 | 评论 / 批注 | 选中文本插评论，可解决 / 可回复 |
| 📋 | 🟡 P2 | Workspace + 成员 | 真正的多账号文档共享（不再依赖 share token） |
| 📋 | 🟡 P2 | 角色与 ACL | viewer / editor / admin |
| 📋 | ⚪ P3 | OAuth (GitHub / Google) | 邮箱密码之外的登录选项 |
| 📋 | ⚪ P3 | SSO (OIDC) | 企业版能力 |
